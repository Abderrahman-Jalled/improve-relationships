# Contributing

Thanks for your interest in improving this skill. Here's how to contribute.

## What belongs here

This repo is a **skill-only** package for OpenClaw. Contributions should improve:

- **SKILL.md** — the core agent-facing instructions
- **Templates** — output scaffolds for digests and outreach
- **Examples** — sample data for testing and demonstration
- **Docs** — product thesis, testing guide, roadmap

## What does not belong here

- Plugin code (planned for v2, separate repo)
- Runtime dependencies or build steps
- Sales CRM features or lead tracking logic

## How to contribute

1. **Fork** the repo and create a branch from `main`
2. **Make your changes** — keep SKILL.md and README.md focused on their respective audiences (agent vs user)
3. **Test** with the prompts in the README and the test cases in `docs/testing.md`
4. **Open a PR** using the pull request template

## Guidelines

- **SKILL.md is agent-facing.** Write instructions the way you'd brief a capable assistant.
- **README.md is user-facing.** Write for a human who wants to know what this does and how to install it.
- **Don't duplicate content** between SKILL.md and README. Each file has one job.
- **Keep examples fictional.** All contacts, signals, and drafts should use made-up data.
- **Respect the draft-only principle.** This skill never sends anything. Every message is a draft.
- **Test cultural occasions carefully.** Check calendar dates. Don't assume backgrounds.
